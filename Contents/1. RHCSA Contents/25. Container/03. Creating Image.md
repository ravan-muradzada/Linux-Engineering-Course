## **What is Creating Image? Why do we need it?**

Sometimes existing images are not proper enough for our purposes. So, we need images with more default functionality. Thus we prefer creating our own images. So, we can use it to create containers on it, afterward.

We always create an image based on another image. So, we actually change or add something to an existing image.

We have a specific file (filename) where we write new image config. It is called `Dockerfile` in Docker and `Containerfile` in Podman.

We use this command to create an image:
```bash
podman build -t <NEW_IMAGE_NAME> /PATH
```
Here, `/PATH` is the place where our `Containerfile` is located in our VM.

## **`Containerfile` Syntax**

I can show the basic syntax of `Containerfile`:

```Containerfile
FROM ubuntu
RUN apt-get update -y
RUN apt-get install nginx
CMD ["sleep" , "5000"]
```

- `FROM` - This line is always the first line and it shows which image we use to create ourselves.
- `RUN` - This basically implements the functionalities in the given image (ubuntu here). We can download, config with command line everything here.
- `CMD` - It is the section where we create a process. It might be a command which run our code.
