## **What is DHCP and DORA?**

DHCP stands for **Dynamic Host Configuration Protocol**. It's a network protocol that automatically assigns IP addresses and other network settings (like subnet mask, default gateway, and DNS servers) to devices on a network. Without DHCP, you'd have to manually configure every device — which is fine for a handful of servers, but impractical at scale.

The **DORA process** is the four-step handshake that happens every time a device requests an IP address from a DHCP server: Discover, Offer, Request, Acknowledge.

---

## **Layer 1: Discover**

In this step the client broadcasts a message access the network saying essentially "Is there any DHCP server available?". Since the client has no IP yet, it broadcasts to 255.255.255.255.

---

## **Layer 2: Offer**

One or more DHCP servers respond this request with an available IP address and some other config details. This is basically "Here is what I can give you".

---

## **Layer 3: Request**

The client picks one of the incoming requests (IPs). (usually the first one received) and broadcasts back a request saying "I accept this offer from this specific server." The broadcast is important — it notifies other DHCP servers that their offers were declined, so they can reclaim those addresses.

---

## **Layer 4: Acknowledge**

The chosen DHCP server confirms the lease, and the client can now use that IP address for the lease duration.

After the lease expires, the process repeats — or the client can send a renewal request before expiry if it wants to keep the same address.
