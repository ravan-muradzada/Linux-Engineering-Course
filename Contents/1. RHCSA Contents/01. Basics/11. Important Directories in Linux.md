#directories

# `/bin`

`/bin` contains **essential user command binaries**. These are basic commands needed for normal users, system boot, rescue mode. Examples: `ls`, `mv`, `cp`, `bash`

👉 `/bin` is actually a symbolic link to `/usr/bin`.

---
# `/sbin`

It contains **administrative commands** mainly used by root. Examples: `mount`, `fsck`, `ip`, `reboot`, `shutdown`.

👉 `/sbin` is actually a symbolic link to `/usr/sbin`.

---
# `/boot`

It contains the files to boot the ststem, like kernel files, bootloader files (like GRUB), initramfs. If `/boot` is broken → system may not start.

---
# `/dev`

It contains device files. These are not normal files. They represent hardware devices.

Inside `/dev` there is a `null/` directory. It is a special file in Linux that **discards everything written to it**. It’s often called the “black hole” of the system. If you send output to `/dev/null`, it disappears permanently.

---
# `/etc`

It contains system config files. If you configure services, you mostly work in `/etc`.

---
# `/media`

Used for **automatic mount** of removable devices.

---
# `/mnt`

Used for **temporary manual mounts**.
