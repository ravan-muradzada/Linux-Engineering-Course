## **What is groups in Linux?**

In Linux, a **group** is a way to organize users so you can manage permissions more easily. Instead of giving file access to users one by one, you assign permissions to a group, and any user in that group gets those permissions. 

Every user has a **primary group** (defined in `/etc/passwd`) and can also belong to multiple **secondary (supplementary) groups** (stored in `/etc/group`). 
Primary group is usually created automatically when you create a user, while secondary groups are used to give extra access, like allowing someone to use `sudo` through the `wheel` group. 

Groups themselves do not give power automatically — they only matter when permissions or configurations reference them.

There are some files which include group related information:
- `/etc/group` → basic group information  
- `/etc/gshadow` → secure group information (like encrypted group passwords)

## **Creating Groups**

We can create groups and set them as primary or secondary groups for users:

```bash
groupadd <GROUPNAME> #Creating a new group
gpasswd groupname #Setting password for a group
usermod -g <GROUPNAME> <USERNAME> #Changing user's primary group
usermod -aG <GROUPNAME> <USERNAME> #Adding user to a new secondary group
groups <USERNAME> #Shows user's all group memberships
```

Using `id` command we can see get data about user himself and its groups:

```bash
id <USERNAME>
```

The result will be like:
`uid=1000(<USERNAME>) gid=1001(<PRIMARY_GROUP_NAME>) groups=1001(SECONDARY_GROUP_NAME)`.
