#systemd #targets

## **What is Target?**

A target is basically a **group of [[02. Units|units]]** that represents a system state.

Think of it like a milestone. When systemd boots, it goes through several milestones — networking is up, filesystems are mounted, system is fully ready. Each milestone is a target.

For example `multi-user.target` means — "network is up, multiple users can log in, system is ready to work." That's the normal state of a RHEL server.

When you enable a service with `WantedBy=multi-user.target`, you're saying — "when this milestone is reached, start me too."

---

## **Commands**

We can see which target your system boots into by default:

```bash
systemctl get-default
```

On a RHEL server it will say `multi-user.target`. On a desktop with GUI it would say `graphical.target`.

We can also change it:

```bash
systemctl set-default multi-user.target
```

---
## **More**

The main line in targets is `AllowIsolate=yes` which represents whether we can enter this target or not. We can imagine targets as environments. 

Some targets depend on other targets. We can think it as stairs. To reach an upper stair, we need to pass the lower ones.

To see all dependencies of any target, we can run this command:

```bash
systemctl list-dependencies multi-user.target
#This shows everything that `multi-user.target` depends on, in a tree format.
```
