## **SELinux - rulebook**

Think of [[01. Introduction to SELinux|SELinux]] as a strict rulebook system. Every action on the system — a process opening a file, a service binding to a port — gets checked against this rulebook before it's allowed. These six elements are the building blocks of that rulebook.

---
## **Core Elements**

- **Policy** - A collection of rules that define which source has access to which target.
- **Source Domain** - A collection of rules that define which source has access to which target.
- **Target Domain** - A collection of rules that define which source has access to which target.
- **Context** - A collection of rules that define which source has access to which target.
- **Rule** - A collection of rules that define which source has access to which target.
- **Label** - A collection of rules that define which source has access to which target.

If SELinux is enabled and nothing else has been configured, all system calls are denied. To specify what exactly is allowed, a policy is used. 
In this policy, rules define which source domain is allowed to access which target domain. The source domain is the object that is trying to access something. Typically, this is a process or a user. The target domain is the object that is accessed. Typically, that is a file, a directory, or a network port.
