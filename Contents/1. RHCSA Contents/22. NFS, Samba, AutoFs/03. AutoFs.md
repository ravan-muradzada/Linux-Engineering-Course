#autofs #packages 

## **What is AutoFs?**

AutoFS is a service that **automatically mounts a share when you access it**, and **unmounts it automatically after a period of inactivity**. Instead of mounting everything at boot, AutoFS waits until something actually tries to use the path.

Think of "Mount whenever a user changes his directory (`cd`) to specific directory, or switches to a specific user".

----
## **Server Side**

#### Package

Install required packages:
```bash
dnf install -y autofs
dnf install -y nfs-utils 
```

#### Firewall

We need to add these services to firewall:

```bash
firewall-cmd --permanent --add-service=nfs
firewall-cmd --permanent --add-service=rpc-bind
firewall-cmd --permanent --add-service=mountd
firewall-cmd --reload
```

#### Config Shared Directory

We need to go to `/etc/exports` file and add the line like that:

```conf
/auto     *(rw)
```

#### Enable Service

We need to start and enable the service of autofs:

```bash
systemctl enable --now autofs
```

----
## **Client Side**

#### Package

Install required packages:
```bash
dnf install -y autofs
dnf install -y nfs-utils #Not required, but recommended
```

#### Master File Config

We should go to `/etc/auto.master` file. Here we find the line starting with `/misc` and write this below:
```
<SHARED_DIR_PATH>    /etc/auto.<ANY_NAME>
```

The second part should start with `/etc/auto.` and last part might be anything like `rhcsa`.

Then we need to create this file:

```bash
vim /etc/auto.rhcsa
```

Write this inside:

```
idtech  -rw,sync  192.168.88.160:/auto
```

#### Enable Service

We need to start and enable the service of autofs:

```bash
systemctl enable --now autofs
```
