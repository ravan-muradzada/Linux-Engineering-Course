#process #commands 

`kill` sends a **signal** to a [[01. Introduction to Process|process]] by PID. The number is the signal type.

We can see all `kill` options by running `kill -l` command.

**`kill -15` (SIGTERM)** — the polite request. "Please shut down cleanly." The process can catch this signal, finish what it's doing, and exit gracefully. This is the default — running just `kill <PID>` sends -15 automatically.

**`kill -9` (SIGKILL)** — the force kill. The kernel terminates it immediately, the process has zero chance to respond or clean up. Use this when -15 doesn't work.

**`kill -1` (SIGHUP)** — originally meant "terminal disconnected." Today it's commonly used to tell a process to **reload its configuration** without fully restarting. Very common with services like nginx or sshd.

**Ctrl+C** — sends **SIGINT (signal 2)** to the foreground process. It's an interrupt request, similar to -15 but triggered from keyboard. The process can also catch and ignore it.

---

## **How to Kill Zombie Process?**

To kill a [[06. Process States|zombie process]], you can either send a `SIGCHLD` (`kill -17`) signal to its parent process or restart the parent service, which will clean up the zombie.
