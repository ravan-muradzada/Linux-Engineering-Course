#package-managers #commands 

## **Subscription**

In RHEL, a **subscription** is basically a **license + access permission** that lets your system receive official software and updates from the Red Hat servers.

Without a subscription, the system still works, but it **cannot access Red Hat repositories on the internet**.

What the subscription gives you:

- Access to **official RHEL repositories** (BaseOS, AppStream, etc.)
- **Security updates and bug fixes**
- **New package versions**
- Support from **Red Hat**

The system is usually registered with Red Hat using:

```bash
subscription-manager register
```

After registration and attaching a subscription, the system can use Red Hat’s online repos and install packages with `dnf`.

Example workflow:

1. Register system
2. Attach subscription
3. Enable repositories
4. Install/update packages with `dnf`

Without a subscription:

- `dnf` cannot download packages from Red Hat servers.
- That’s why in many labs (like yours) we **mount the ISO and create a local repo** from `BaseOS` and `AppStream`.

-----

## **Config?**

If you **register and attach a subscription successfully**, normally you **don’t need extra repo configuration**. The system will automatically enable the official Red Hat repositories and `dnf`/`yum` can install packages from the internet.

---

## **Unsubscribe**

If a system was previously registered, you can remove it with:

```bash
sudo subscription-manager unregister
```
