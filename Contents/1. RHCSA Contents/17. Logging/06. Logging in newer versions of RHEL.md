#logging #rhel-10 

In newer [[02. Introduction to Red Hat|RHEL]] versions (like RHEL 8 and RHEL 9), [[01. Introduction to Logging|logging]] is centered around **[[02. Logging Approaches and Destinations|systemd-journald]]**. This service collects almost all system logs first, including [[01. Kernel Basics|kernel]] messages, service outputs, [[01. Introduction to Boot Process in Linux|boot]] information, and authentication events. These logs are stored in [[05. Journald and Journalctl Command|journald]]’s own database format, which you normally access using `journalctl`. If persistent logging is enabled, they are saved under `/var/log/journal/`.

At the same time, **rsyslog** is still commonly used, but it works more like an optional processing layer. Instead of being the primary collector, rsyslog can read logs from journald and then write selected messages into traditional text files such as `/var/log/messages` or `/var/log/secure`. This is mainly done for compatibility, reporting, or integration with other tools.

Because of this design, logs are not simply “moved” from rsyslog to journald or vice versa. Journald is always collecting everything at the system level, while rsyslog only handles what it is configured to process and store in file form or forward to external systems.

So in modern RHEL systems, journald is the main source of log data, and rsyslog acts as a flexible exporter that can turn parts of that data into traditional log files or send them to remote servers.
