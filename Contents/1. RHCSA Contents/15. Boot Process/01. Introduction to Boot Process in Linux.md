## **Intro to Boot Process in Linux**

Many processes are running in the background when we press the system's power button. It is very important to learn the Linux boot process to understand the workings of any operating system. Knowing how the kernel boots is a must to solve the booting error.

---

## **Booting Flow**

![[012 - Boot Process Flow.png]]

1. **Firmware (BIOS/UEFI)** — the very first thing. The hardware runs a self-test (POST), initializes devices, then looks for a bootable disk based on the boot order. UEFI is the modern version; it reads the EFI System Partition directly.
2. **MBR (Master Boot Record)** — the very first 512 bytes of the disk. It contains a tiny piece of code whose only job is "find and start the bootloader." In modern systems, UEFI + GPT replaces this, but MBR still exists on older setups. GRUB2 actually installs itself _into_ the MBR on BIOS systems.
3. **GRUB2** — the bootloader in Linux. It shows you the boot menu, then loads the kernel image (`vmlinuz`) and the `initramfs` archive into memory. It also passes kernel parameters (like `rd.break` which you've used for recovery).
4. **KERNEL** — the Linux kernel starts, detects hardware, and sets up the system. In modern RHEL, it also unpacks initramfs here before continuing — the old simplified flow skips this detail.
5. **Systemd / INIT** — in the old world, `init` was the first process (PID 1), and it used a config file called `/etc/inittab` to decide what to start. RHEL 7 replaced `init` with `systemd`. Same role, completely different approach. systemd is faster because it starts services in parallel; old init did it one by one.
6. **Target / Runlevel** — this is the old SysV term. A runlevel was just a number (0–6) that told init _which mode_ to boot into. systemd replaced runlevels with **targets** — but kept the old names as aliases so nothing breaks.

## **More**

The files about booting process are located under `/boot` directory. 

Main GRUB config file is `/boot/grub2/grub.cfg`. However, we never modify this file. 
We modify `/etc/default/grub` file and it affects the previous file I mentioned. Contents of this file is like that:

```
GRUB_TIMEOUT=5
GRUB_DISTRIBUTOR="$(sed 's, release .*$,,g' /etc/system-release)"
GRUB_DEFAULT=saved
GRUB_DISABLE_SUBMENU=true
GRUB_TERMINAL_OUTPUT="console"
GRUB_CMDLINE_LINUX="crashkernel=1G-2G:192M,2G-64G:256M,64G-:512M rd.lvm.lv=rhel/root"
GRUB_DISABLE_RECOVERY="true"
GRUB_ENABLE_BLSCFG=true
```

We mostly change only the first line which defines the seconds to pass GRUB menu automatically. 

After making changes in this file, we need to [[01. Redirections|redirect]] its contents to `/boot/grub2/grub.cfg` file (of course with `>>`):

```bash
grub2-mkconfig >> /boot/grub2/grub.cfg
#grub2-mkconfig is a specific command
```
