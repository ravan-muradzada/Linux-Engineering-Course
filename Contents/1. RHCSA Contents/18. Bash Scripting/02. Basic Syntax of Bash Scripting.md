## **`if`**

In bash, `if` checks whether something is **true or false**, then acts accordingly.

---

**Basic syntax:**

```bash
if [ condition ]; then
    # commands
elif [ condition ]; then
    # commands
else
    # commands
fi
```

`fi` just closes the `if` block (it's "if" backwards).

---

## **When to use `[]`**

You use `[ ]` when you're testing something — comparing numbers, checking strings, checking if a file exists, etc.

```bash
if [ $age -gt 18 ]; then
    echo "Adult"
fi
```

You **don't** use `[ ]` when you're directly running a command and checking if it succeeded or failed:

```bash
if systemctl is-active nginx; then
    echo "nginx is running"
fi
```

Here, the command itself returns success (0) or failure (non-zero) — bash uses that directly. No brackets needed.

**The core rule:** `[ ]` is for **tests/comparisons**. No brackets when you're running a **real command** and trusting its exit code.

---
## **Common test operators:**

For numbers: `-eq`, `-ne`, `-gt`, `-lt`, `-ge`, `-le` For strings: `=`, `!=`, `-z` (empty), `-n` (not empty) For files: `-f` (file exists), `-d` (directory exists), `-x` (executable)

```bash
if [ -f /etc/passwd ]; then
    echo "File exists"
fi
```

---
## **Important**

One thing to never forget — **spaces inside the brackets are mandatory:**

```bash
[ $x -eq 5 ]   # correct
[$x -eq 5]     # error
```
