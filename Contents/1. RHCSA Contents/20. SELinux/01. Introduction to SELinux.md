## **What is SELinux?**

SELinux (Security-Enhanced Linux) is an extra security layer built into the [[01. Introduction to Linux|Linux]] [[01. Kernel Basics|kernel]]. Think of regular Linux [[01. Introduction to Permissions|permissions]] (rwx) as a bouncer checking IDs at the door — SELinux is a second bouncer _inside_ who checks what each person can actually do once they're in.

It was originally developed by the NSA and is mandatory on [[02. Introduction to Red Hat|RHEL]] systems. You can't just remove it — and for RHCSA, you need to work with it, not around it.

> [!Info] **How it works?**
> SELinux attaches labels to every file, process, and port, then enforces rules about which labels can interact with which.

---
## **Modes of SELinux**

**There are 2 modes of SELinux**:

- **Active**: Here SELinux is active and do some process. It has two categories:
	- **Enforcing** — SELinux is fully active. It blocks and [[01. Introduction to Logging|logs]] policy violations. This is the default on RHEL and the production-ready mode.
	- **Permissive** — SELinux doesn't block anything, but still logs violations. Useful for _troubleshooting_ — you can see what _would_ have been blocked without breaking things.
- **Disabled** — SELinux is completely off. Strongly discouraged on RHEL, and switching between disabled and the other modes requires a reboot plus filesystem relabel.

---
## **Switching Between Modes**

There are two kinds of change between SELinux modes: temporary and permanent.

#### **Temporary change** (lost after reboot):

```bash
setenforce 0    # Enforcing → Permissive
setenforce 1    # Permissive → Enforcing
```

Note: you cannot use `setenforce` to switch to Disabled mode — that's permanent-only.

Verify with `getenforce` after switching.
#### **Permanent change** (survives reboot):

Edit this file:

```bash
vi /etc/selinux/config
```

Find and change this line:

```
SELINUX=enforcing    # options: enforcing, permissive, disabled
```

Save and reboot. The change takes effect on next boot.


> [!Warning] Modes of SELinux
> If you switch _from_ Disabled to Enforcing/Permissive, the system needs to relabel the entire filesystem on next boot — this happens automatically but takes extra time. That's why going Disabled is discouraged; it's not a quick toggle.
> 
> A practical tip for RHCSA: if something is broken and SELinux looks suspicious, switch to Permissive temporarily with `setenforce 0`, test, then fix the real SELinux issue rather than disabling it permanently. Examiners notice that.

## **Error Logs Of SELinux**

Error [[02. Logging Approaches and Destinations|logs of SELinux]] is located inside `/var/audit/audit.log`. 
We use this command to see errors in log file:

```bash
grep AVC /var/log/audit/audit.log
```

`AVC` means error logs.
