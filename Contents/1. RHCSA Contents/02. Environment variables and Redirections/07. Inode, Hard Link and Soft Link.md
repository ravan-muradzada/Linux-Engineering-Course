## **What is inode?**

An **inode** (index node) is a data structure in Linux file systems that stores **information about a file**, but not the file name and not the actual data itself. 

It stores:
- File type (regular file, directory and etc)
- File permissions
- Hard link count
- Owner and group 
- File size
- Timestamps (created, updated time)
- Pointers to the disk blocks where the real data is stored

We can see inode numbers using `ls -i`. Here the first number is inode number.

When we create a file, inode is created for this file. So the file itself has connection with inode and inode has another connection with disk blocks (clusters).

----

## **What is Hard Link?**

- When we create a hard link for a file, the link is connected directly to inode of the main file. If we create multiple hard links, all are connected directly inode of the main file.
- We can create hard links for a file. **We cannot create a hard link for directory!**
- So, that is why when we check inode-s of files using `ls -i`, we will see inodes of the hard links and main file are the same. 

![[001 - hard link.png]]


- Creating hard links does not require more disk space from the device, they are just references.
- Removing any hard link or even the main file, other files stay the same. Only with removing all hard links and main file, the data will be removed.
- Modifiying any file among them will modify others, too.

Syntax:

```bash
ln main_file.txt hardlink_1
ln main_file.txt hardlink_2
ln main_file.txt hardlink_3
#First name of main file, then hard link name
```

----

## **What is Soft (Symbolic) Link?**

- When we create a soft link for a file, a new inode is created for this soft link and its inode is connected to inode of the main file.
- We can create soft links for both files and directories.
- So, that is why when we check inode-s of files using `ls -i`, we will see inodes of the soft links and main file are different.

![[002 - Soft link.png]]

- Creating soft links does not require more disk space from the device, they are just references.
- Removing any soft link does not affect other soft links or the main file. However if we remove the main file, all soft links become meaningless.
- Modifiying any file among soft links and main file will modify others, too.
- We can create soft link for directories, too. In that case, each file in main directory is created inside the soft link, too. Any changes in soft link directory and its files will happen in main directory, too. In addition, changes in main directory affect soft link directory, too.

Syntax:
```bash
ln -s main_file.txt softlink_1
ln -s main_file.txt softlink_2
ln -s main_file.txt softlink_3
#First name of main file, then soft link name
```
