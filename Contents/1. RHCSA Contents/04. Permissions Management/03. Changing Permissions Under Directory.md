## **Changing Permission Under Directory**

`chmod -R` means “change permissions **recursively**.”

The `-R` option stands for _recursive_. It tells `chmod` to apply the permission changes not only to the given directory, but also to **all files and subdirectories inside it**, and everything inside them.

For example:

- `chmod -R u+x dir1`  
    → adds execute permission for the owner to `dir1` and everything inside it.
    
- `chmod -R go-w project`  
    → removes write permission for group and others from the directory `project` and all its content.

---

## **Not Giving Execute to Regular Files**

When we write `chmod -R a+x myDir`, it will give **execute** permissions to all regular files and directories under the directory. However it may be dangerous or bad practise to give or take **execute** permissions to all regular files. 

That is why instead of `x`, we use `X` for **execute** permissions in the case.

`X` (capital X) is smarter. It means “set execute permission only if it makes sense.” Specifically, it adds execute permission **only to directories**, and to files that already have execute permission for at least one user. It does not add execute permission to regular non-executable files.

Example:

If you run `chmod -R a+X project`:

- All directories inside `project` will get execute permission (so users can enter them).
- Files that are already executable (like scripts) will stay executable.
- Normal files (like `.txt`) will NOT become executable.

This is why `X` is commonly used when fixing permissions after copying files. It prevents accidentally turning every file into an executable file.
