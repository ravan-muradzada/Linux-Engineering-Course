#systemctl #commands 

## **What is Systemctl Command for?**

`systemctl` is the **main command to talk to [[01. Introduction to Daemons and Systemd|systemd]]** — to manage [[02. Units|units]], check their state, and control system behavior.

We can use these with `systemctl`: 
`status`, `start`, `stop`, `enable`, `disable`, `restart`, `reload`, `is-active`, `is-enabled`, `mask`

----
## **Commands**

### **`status`**

It shows if the service is running, its last logs, PID, and any errors. First command you run when something breaks.

![[011 - Systemctl Command.png]]
**Loaded line:**

- `loaded` — systemd read the unit file successfully
- `/usr/lib/systemd/system/sshd.service` — where the unit file lives
- `enabled` — will start at boot. If it is `disabled`, even if it is currently running, it will not run when we reboot the system.
- `preset: enabled` — the vendor/Red Hat default for this service is also enabled

**Active line:**

- `active (running)` — currently running right now. It might be `active (exited)` for one time tasks, `active (waiting)` which shows the unit is running and waiting for an event and `inactive (dead)` which shows the unit is not running.
- `since Tue 2026-04-07 01:21:18` — when it started
- `15eh ago` — how long it's been running


### **`enable` and `disable`**

- `enable` creates the symlink, so service starts at boot. Doesn't start it right now. 
- `disable` removes this smlink, so it will not start at boot.

```bash
systemctl enable httpd
systemctl disable httpd
```

If we write `systemctl enable --now httpd`, it both enable and activate the service, so it is also running right now.


### **`start` and `stop`**

- `start` starts the service **right now**. Doesn't survive reboot.
- `stop`  stops the service right now.

```bash
systemctl start httpd
systemctl stop httpd
```


### **`restart` and `reload`**

- `restart` — stop + start together. Use when you changed a config and need full restart.
- `reload` — tells the service to re-read its config **without stopping**. Faster and safer than restart, but not all services support it.

```bash
systemctl restart httpd
systemctl reload httpd
```

### **`is-active` and `is-enabled`**

- `is-active` just checks whether the service is active or not, returns `active` or `inactive`. Good for scripts.
- `is-enabled` checks if the service is enabled or not, returns `enabled` or `disabled`. Checks boot behavior, not current state.

```bash
systemctl is-active httpd
systemctl is-enabled httpd
```

### **`mask` and `unmask`** 

- `mask` — **completely blocks** the service. Even `systemctl start` will fail. Stronger than disable. Use when you want to make sure something never runs. (If in our system there is `http` running and we block `nginx` by default).
- `unmask` — reverses mask.

```bash
systemctl mask httpd
systemctl unmask httpd
```
