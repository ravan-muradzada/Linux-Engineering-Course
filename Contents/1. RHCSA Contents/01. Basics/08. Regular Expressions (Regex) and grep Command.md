#regex #commands 

## **What is Regex?**

A **regular expression** is a **pattern** used to search, match, or filter text. It is not a command. It is a rule for describing text patterns. We may think it as a smart search formula.

----

## **Grep Command**

The `grep` command is a powerful utility for searching text using (regex) patterns. It searches for PATTERNS in each (mentioned) FILE and prints each line that matches a pattern. Typically, PATTERNS should be quoted when `grep` is used in a shell command.

Syntax:
```bash
grep [OPTION...] PATTERNS [FILE...]
```

Some important options:

`-i`: Ignore case distinctions in patterns.
`-w`: Select only those lines containing matches that form whole words.
`-c`: Suppress normal output, instead print a count of matching lines for each input file.

---

## **Regex Usage**

```bash
grep hello file.txt #Without regex. Find the lines with hello.
grep `^hello` file.txt #Finds the lines starting with hello.
grep `h.llo` file.txt #Finds the lines with h.llo where . means any one symbol.
grep `h[au]llo` file.txt #Finds the lines with hallo and hullo.
grep `h[^au]llo` file.txt #Finds the lines with matching `h.llo` except a and u.
grep `hello$` file.txt #Finds the lines ending with hello.
```
