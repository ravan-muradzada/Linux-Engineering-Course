#network #commands #configuration

**nmtui** (NetworkManager Text User Interface) is a simple terminal-based menu tool that lets you configure network settings visually — without needing to remember command syntax. It's a friendlier front-end to NetworkManager.

```bash
sudo nmtui
```

It gives you three options: **Edit a connection** (create or modify network interfaces, set static IPs, DNS, gateway, etc.), **Activate a connection** (bring a connection up or down), and **Set system hostname** (change the machine's hostname).

## **Edit Connection**

If we click **Edit Connection**, it will show us available connections in the system like that:
![[005 - nmtui2.png]]

If we want to edit any connection config, we need to select it and then go to **Edit**.

![[006 - nmtui3.png]]
As you can see here, we change a lot of config here:
- We can change **Profile Name** part.
- We should not change **Device** part.
- In IPv4 Configuration part we can convert it from **Automatic** to **Manual**. In **Automatic** mode, private ip is automatically given. **Manual** means we ourselves give ip to our machine manually. However, we need to be attentive, because if two private ips are the same in the same network, both of them will now work properly.
- Below IPv4 Config we can change our **Address** (our [[01. IP Address and Its Variations|private ip]]), **Gateway** and more.
- Below IPv4 Config we can change or add DNS servers, too. It is good to add `8.8.8.8` and `8.8.4.4` by default.
- If in our country IPv6 is not used, we can disable it.
- Automatically Connect part is important, because if we disable it when we connect to our machine, it will not connect to network automatically.

## **Activate Connection**

In this section, we can activate or deactivate any connection. If we make any changes on any connection by using **Edit Connection**, we need to deactivate and then activate this connection. Otherwise the changes will not be seen.

## **Multiple Adapters**

We can add multiple adapters to have multiple private IPs at the same time. In [[04. Installation and configuration of Red Hat 9.7|VMWare Workstation Pro]] it is easy to add new network adapters. We just need to right click to our virtual machine and choose `Settings` here. Then click `Add > Network Adapter > Finish > Ok`.

After adding a new adapter, when we run `ip a` command it will show all network adapters available in the system. 

Multiple adapters help us when our virtual machine has multiple applications inside. For each application one private ip is used. 

When we use `nmtui` command, but our adapter is not in the list, we need to create it manually by clicking `Edit Connection > Add > Ethernet`. Then we can give any profile name, however in `Device` section we need to be attentive. When we run `ip a`, it shows [[03. MAC Address|MAC Adress]] of a new adapter, we need to copy it. Then in `Device` section, we write `ens224 (<MAC_ADDRESS>)` if its profile name is `ens224`.
