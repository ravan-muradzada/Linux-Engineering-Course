## **What is a disk?** 

A disk is a storage device where your OS, files, and data live. In Linux, every disk is represented as a file under `/dev/` — because in Linux, everything is a file.
##### **Main disk types you'll see:**

`/dev/sda`, `/dev/sdb` — these are physical or virtual hard drives (HDD or SSD). The letter changes for each disk (sda = first, sdb = second, etc.).

`/dev/nvme0n1` — NVMe SSDs, much faster than regular SSDs. Common in modern machines.

`/dev/vda` — virtual disks, common in VMs (like KVM/QEMU). You'll see this a lot in your RHEL labs.

---
## **HDD vs SSD**

**HDD vs SSD — does it matter in Linux?** From a sysadmin perspective, you manage them the same way — same commands, same partitioning tools. The difference is speed and hardware internals, not how Linux handles them.

**One thing to know early:** when you add a partition to a disk, it gets a number — `/dev/sda1`, `/dev/sda2`, etc. So the disk is `sda`, and its partitions are `sda1`, `sda2`, and so on.

---
## **How to add more disk to VM?**

We need to right click to our VM and select Settings there. Then we need to add hard disk there. There are a window where it is asked to choose virtual disk type. If we choose `SCSI`, that means HDD. If it is `NVMe`, it means SSD.

> [!Warning] Important Cue
> When we add a new disk space to our existing virtual machine and if it is SSD, we need to restart it to see. However, if it is HDD, it is not required.
