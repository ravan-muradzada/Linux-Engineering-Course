#crontab 

## **What is Cron Deny?**

Crontab access is controlled by two files: `/etc/cron.allow` and `/etc/cron.deny`.
The `cron.deny` file is a blacklist — you list usernames one per line and those users are blocked from using crontab, while everyone else can still use it freely, for example adding `john` and `mary` to `/etc/cron.deny` blocks only them. 

----
## **What is Cron Allow?**

The `cron.allow` file is a whitelist — only the users listed inside can use crontab, and everyone else is automatically blocked.

The critical rule is that **`cron.allow` completely overrides `cron.deny`** — if `cron.allow` exists, cron doesn't even look at `cron.deny`. 

The most powerful lockdown is creating an empty `cron.allow`:

```bash
touch /etc/cron.allow
```

Since nobody is listed, nobody gets access, not even regular users. In practice, on a secure server you'd use `cron.allow` to whitelist only the specific users or service accounts that genuinely need scheduled jobs, rather than trying to maintain a growing blacklist in `cron.deny`.
