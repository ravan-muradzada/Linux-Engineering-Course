#samba #packages 

## **What is Samba?**

Samba is the Linux implementation of SMB (Server Message Block) — Microsoft's file sharing protocol.

Samba is the [[01. Introduction to Linux|Linux]] implementation of **SMB (Server Message Block)** — Microsoft's file sharing protocol. While [[01. NFS|NFS]] is Linux-to-Linux, Samba is how Linux shares files with **Windows machines** (and vice versa).

The practical difference is that Windows natively understands SMB, so when you set up Samba on a Linux server, a Windows client sees it just like a regular Windows network share — no extra software needed on the Windows side.

## **Server Side**

#### Package

We should download its package:
```bash
dnf install samba -y
```

#### Firewall and SELinux

We need to add its [[03. Systemd Services|service]] file to [[04. Firewall|firewall]]:
```bash
firewall-cmd --add-service=samba --permanent
firewall-cmd --reload
```

We need to change context type of this directory for SELinux:

```bash
semanage fcontext -a -t samba_share_t  "/<DIR_NAME>(/.*)?"
restorecon -R -v <DIR_NAME>
```
#### Sharing Config

We need to go this file to make configs: `/etc/samba/smb.conf`. Then we add our own sharings. (there are some, by default)

We add this to end of the config file:

```bash
[sambashared]
comment = sambashare
read only = No
browseable = Yes
path = /<SHARED_DIR_PATH>
```

#### Samba User

It is required to have a user for `samba` in Linux. In windows, we will use this user's credentials to access the shared directory.

```bash
useradd samba-user
smbpasswd -a samba-user #Setting passwd
```

#### Service Start (and enable)

Then we need to enabe and start the service of `smb`:

```bash
systemctl enable --now smb
```

---
## **Client Side**

In windows, we just need to press `Windows+R` and enter `\\<SERVER_IP>`. Then enter samba user's credentials to access the directory.
