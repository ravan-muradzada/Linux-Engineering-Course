## **`top` Command**

`top` is a **live, real-time** version of `ps` — it refreshes every few seconds and shows [[01. Introduction to Process|processes]] sorted by CPU usage by default. While `ps` [[03. ps Command|takes]] a snapshot and exits, `top` stays open and keeps updating.

The key fields are the same (PID, USER, %CPU, %MEM, VSZ, RSS) but you also get a **summary header** at the top showing overall CPU, memory, load average, and how many tasks are running/sleeping.

![[008 - top command.png]]

There are some essential fields before listing the process when we run `top` command:
- The first line starts with time information. Then we can the number of users in the systems. At `load average` part we see overload in the system within `1`, `5` and `15` minutes.
- The second line gives information about the count and state of processes.
- The third line represents CPU jobs and its overload.
- The fourth line shows total, free, user and cached RAM. Important: **cache is NOT a problem**, Linux uses it smartly.
- The fifth line gives info about swap memory.

There are some fields for the list of process which give us detailed information about them:
- `PID` is ID of process.
- `USER` shows the starter of the process.
- `PR` is kernel-assigned priority of the process. It is between `0` and `39`. Lower number is higher priority. It shows how “important” the process is to CPU scheduling.
- `NI` is user-adjustable part of priority. It is between `-20` and `19`. Lower number is higher priority. `PR` = `NI` + 20. *A process with `NI=0` and `PR=20` → normal priority.*
- `VIRT` is total virtual memory(everything the process _can_ access: RAM + swap + mapped files).
- `RES` is real memory (RAM actually used now). This is what you care about most.
- `SHR` is shared memory (part of RES that is shared with other processes, like libraries).
- `S` is the state of process.
- `%CPU` is percentage for CPU usage.
- `%MEM` is percentage for RAM usage.
- `TIME+` is **total CPU time the process has used since it started** — **including all threads**, in **hundredths of a second**.
- `COMMAND` shows which command is used to start the process.

## **`htop` Command**

**`htop`** is just a better, more user-friendly version of `top`. The differences are practical: it's color-coded, you can scroll both vertically and horizontally, you can kill or nice a process directly with function keys, and the header shows CPU/memory as visual bars instead of plain numbers. It's not installed by default on RHEL, you need external [[03. Configuration of Package Manager (DNF)|repositories]].

![[009 - htop command.png]]

- In the leftside:
	- The first two colorful bars represent our **two CPU cores**. (There are 2 available CPU core for this VM).
	- `Mem` bar is our **RAM usage** — you can see it's partially used (those colored bars mean used memory, different colors = kernel/user/cache split).
	- `Swap` shows swap space.
- In the rightside:
	- `Tasks: 43, 15 thr, 194 kthr; 0 running` — 43 processes total, 15 user threads, 194 kernel threads, and **0 actively running** right now (all sleeping).
	- `Load average: 0.10 0.04 0.05` — system load over the **last 1, 5, and 15 minutes**. These numbers are very low, meaning your system is relaxed. On a 2-core system, anything under 2.0 is healthy.
	- `Uptime: 02:02:25` — system has been running for 2 hours.

## **More Info**

If there is a performance problem in our VM, there are steps we need to do for monitoring:
1. `df -h` - It shows how much disk space is used and free in our system.
2. `free -mh` - It shows how much RAM is used and free in our system.
3. `top` or `htop` - It is used to monitor real time processes.
