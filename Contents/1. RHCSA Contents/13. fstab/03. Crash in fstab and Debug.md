## **Problem and Its Reason**

`fstab` causes boot failure when Linux tries to [[05. Mounting|mount]] something at boot but **fails** — it drops you into emergency mode.

**Common causes:**

- Wrong UUID or device name that no longer exists
- Typo in the mount point or filesystem type
- A network drive (NFS) that isn't available at boot time
- Removed a [[01. Introduction to Disk|disk]] but forgot to remove its `fstab` [[02. Syntax of fstab file|entry]]

---

## **How to Solve The Problem?**

When we add something wrong to `fstab` file, it may show us:
- `A start job is running for /dev/disk/by-uuid/MeaningLess_Name (1 min 24 s / no limit)`, we press `Ctrl+C`
- `Give root password for maintenance (or press Ctrl+D to continue)`
- Or something other

We just need to enter root password and enter the system. Then we need to type `vi /etc/fstab` and remove or comment the wrong line. Then we need to reboot our system.

Sometimes, `/etc/fstab` becomes read-only. Now we need to run this command:

```bash
mount -o remount,rw /
```
