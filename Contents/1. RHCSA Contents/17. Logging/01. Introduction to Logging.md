## **What is Logging?**

Logging is simply the habit of recording what your system is doing so you can **see problems, understand behavior, and prove what happened later**. 

At the center of modern [[01. Introduction to Linux|Linux]] logging is **[[01. Introduction to Daemons and Systemd|systemd]]-journald**. It collects logs from the [[01. Kernel Basics|kernel]], services, and applications. You read those logs using **journalctl**.

Here’s the practical idea:

- Every [[03. Systemd Services|service]] (like [[02. Configuring SSH|SSH]], [[01. Introduction to Crontab|cron]], nginx) writes messages
- The system stores them
- You query them when something breaks

Simple example:

- You can’t SSH into a server → check logs → see “permission denied” or “port blocked”

Why logging matters (real talk):

- Without logs, debugging is guessing
- With logs, debugging becomes evidence-based

## **What is ELK - Elastic Search, Logstash, Kibana?**

ELK is a stack used to **collect, store, search, and visualize logs at scale**. Think of it as logging on steroids — instead of checking one server, you monitor **hundreds or thousands** in one place.

ELK = three main tools working together:

- **Elasticsearch** → stores logs and lets you search them very fast
- **Logstash** → collects and processes logs (parses, filters, transforms)
- **Kibana** → shows logs in dashboards, graphs, charts

How it works in real life:

Your servers generate logs → Logstash collects them → sends to Elasticsearch → you view them in Kibana.

Example:

- 50 Linux servers running SSH, nginx, apps
- All logs go to ELK
- In Kibana, you can search: “failed SSH login last 10 minutes”
- Instantly see patterns, [[01. IP Address and Its Variations|IPs]], attacks

That’s the key advantage:  
👉 **centralized logging + powerful search**

Why companies use it:

- Debug faster (no SSH into each server)
- Security monitoring (detect attacks)
- Analytics (traffic, errors, usage)

## **Grafana**

**Grafana** is a tool used to **visualize data** in real time. It doesn’t store logs itself; instead, it connects to data sources like Elasticsearch, Prometheus, or databases and turns raw data into **clear dashboards with graphs, charts, and alerts**. For example, you can see CPU usage, memory, or even failed login attempts across servers in one screen. It’s widely used in monitoring systems because it makes complex data easy to understand quickly, which helps you detect problems faster and react before things break.
