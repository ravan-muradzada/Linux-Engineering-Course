#network #commands #configuration 

**`nmcli`** is a command-line tool used to control **NetworkManager** in systems like **Red Hat Enterprise Linux**.

NetworkManager manages:

- network interfaces
- IP addresses
- Wi-Fi / Ethernet connections
- DNS
- gateways

So **`nmcli` = Network Manager Command Line Interface**.  
It lets you manage networking **from the terminal instead of using `nmtui` or GUI**.

Example things you can do with it:

```bash
nmcli device status
nmcli connection up <name>
nmcli connection down <name>
nmcli connection add ...
```

**`nmcli connection show`**

This command lists **all NetworkManager connection profiles** saved in the system.

Example output:

```
NAME        UUID                                  TYPE      DEVICE
ens160      1c2b4a9c-xxxx-xxxx-xxxx-xxxxxxxxxxxx  ethernet  ens160
Wired 1     a8d0e2c4-xxxx-xxxx-xxxx-xxxxxxxxxxxx  ethernet  --
```

Meaning:

- **NAME** → connection profile name
- **UUID** → unique identifier
- **TYPE** → ethernet, wifi, etc.
- **DEVICE** → which interface is using it

Important idea:

- **Connection profile** = saved network configuration
- **Device** = actual network card (`ens160`, `ens192`, etc.)

So a device may exist (`ip a` shows it), but **no connection profile exists for it**, which is exactly the problem you saw earlier.

You can also inspect a specific connection:

```bash
nmcli connection show ens160
```

This prints full details:

- IP address
- gateway
- DNS
- method (manual/DHCP)

A good mental model:

- `ip a` → kernel network interfaces
- `nmcli device` → devices known by NetworkManager
- `nmcli connection` → **saved configurations**
