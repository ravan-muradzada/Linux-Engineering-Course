#redirections

## **What is Redirection?**

It is basically redirecting process of the output of any command to a file and save it here. It has multiple versions: `>` (`1>`), `>>`(`1>>`), `2>`, `2>>`, `&>`, `<`.

---
## **Stdin, Stdout, Stderr**

- **stdin** is number 0. This is where a program **receives input**. Normally, it comes from your keyboard.
- **stdout** is number 1. This is the **normal output** of a command. By default, it prints to your terminal.
- **stderr** is number 2. This is for **error messages only**.

----

## **`>` and `>>`**

- `>` redirects **stdout** to a file. If the file exists, it **overwrites** it. Same with `1>`.
- `>>` redirects **stdout** but **appends** to the file. It does NOT delete old content. Same with `1>>`.

---

## **`2>` and `2>>`**

- `2>` redirects **stderr** to a file. Overwrites the file.
- `2>>` redirects **stderr** to a file. It does NOT delete old content, just appends.

> [!Info] Combination of different redirections
> ```bash
>ll /existingDir /nonExistingDir 1>success.txt 2>fail.txt
> ```
> `ll` commands searches for both directory. For existing one, it redirects **stdout** to `success.txt`, for the other to `fail.txt`.
> 
> Moreover, sometimes we can redirect errors to `/dev/null` to discard them.



---
## `&>` and `&>>`

- `&>` redirects **both stdout and stderr** to the same file. Overwrites. `ls &> file.txt` and `noCommand &> file.txt`. Both are redirected to `file.txt`.
- `&>>` redirects **both stdout and stderr** to the same file, too. It does not overwrite, just appends.

---
## `<`

It redirects **input** into a command (stdin). 
Example:  
`sort < file.txt`  
Instead of typing input manually, it reads from file.txt.
