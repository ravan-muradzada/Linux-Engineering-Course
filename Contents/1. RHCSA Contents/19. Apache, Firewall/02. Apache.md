## **What is Apache?**

Apache (full name: **Apache HTTP Server**) is one of the oldest and most widely used web [[01. Introduction to Web and Application Servers|servers]] — been around since 1995.

It works in a **[[01. Introduction to Process|process]]/thread-based** model, meaning for each incoming connection it spawns a process or thread to handle it. This works fine but under very high traffic it consumes a lot of memory because you end up with hundreds of processes running simultaneously.

This is actually why **Nginx was created** — to solve Apache's performance problem under heavy load. Nginx uses an event-driven model (handles many connections in one thread), which is much more memory-efficient.

**So is Apache dead?** No. It's still widely used, especially in shared hosting environments (cPanel, for example, runs Apache). It also has a feature called **.htaccess** — per-directory configuration files — which Nginx doesn't have. This makes Apache more flexible for shared hosting where different users need different configs without touching the main server config.

**Apache vs Nginx simply:**

Apache is more flexible and forgiving to configure. Nginx is faster and lighter under high load. In modern DevOps stacks, Nginx wins in most cases. But you'll still encounter Apache in legacy systems and hosting environments, so knowing it matters.

## **Installation**

Apache package in Linux (RHEL) is called `httpd`. So, we download it like that:

```bash
dnf install httpd -y
```

Its [[03. Systemd Services|service]] comes to our machine as disabled service. So, we need to both [[07. Systemctl Command|enable]] and start it:

```bash
systemctl enable httpd
systemctl start httpd
```

---
## **Getting Access**

We can get an access to its default `index.html` from the browser by entering our [[01. IP Address and Its Variations|IP]] in search bar. However, we need to stop `firewalld` service beforehand. (We will learn its config later)

```bash
systemctl stop firewalld
```

## **Config File**

Main config file for apache is `/etc/httpd/conf/httpd.conf`. It is modular and the directory for modules: `/etc/httpd/conf.modules.d`.

If we make any change in apache config file, we need to either turn off SELinux or config it, too. Moreover, we need to restart `httpd` server.

#### Port and IP
There is a specific line with:

```
#Listen 12.34.56.78:80
Listen 80
```

Here we can see that the default port for apache is `80`. Moreover, from the first line here we can define which network adapter may have an access to apache.

#### Denied and Allowed Directories

There is a specific block inside the main config file:
```
<Directory />
    AllowOverride none
    Require all denied
</Directory>
```
Here, `/` directory (so, every directory) is denied for apache to use.

Then there is another block like that:
```bash
<Directory "/var/www">
    AllowOverride None
    # Allow open access:
    Require all granted
</Directory>
```
Here, specific directory `/var/www` is allowed for apache. We can add another blocks like that to add another directory for apache, but it is not recommended to remove the default one.
