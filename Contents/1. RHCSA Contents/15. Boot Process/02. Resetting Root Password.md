#boot-process 

## **Why?**

**The idea** — if we lose the root password, we can interrupt the boot process before the real system starts, drop into a minimal shell, change the password, then continue booting normally. Root reset happens after GRUB finishes and before Kernel starts the process. 
That is very important for [[03. Introduction to Red Hat Certificates|RHCSA]].

---

## **Resetting Root Password - Method 1**

#### 1. Start VM and Stop it in GRUB menu

After opening [[01. Introduction to Boot Process in Linux|GRUB menu]], we need to press `e` from the keyboard to access configuration stuff.

![[013 - GRUB menu.png]]

The configuration part is like that:

![[014 - GRUB menu config.png]]

#### 2. We need to change two parts in config file
 
- `ro` → `rw` ⎯  `ro` refers the system is read-only and we change it to `rw` which is read & write.
- `init=/bin/bash` ⎯ If there is `rhgb quiet`, change it to `init=/bin/bash`. Otherwise add it to the end of the fourth (in the image above,  it is after `rd.lvm.lv=rhel/root`).

After the modification, we need to press `Ctrl+X` to boot. After some time, the [[05. Shell and its varieties|shell]]
 will open.
#### 3. We change root password

We need to run [[01. User Management|this command]] to change root password:

```bash
passwd root
```

Then we need to create a specific file named `/.autorelabel` file in `/`:

```bash
touch /.autorelabel
```

That is about SELinux. Without this, SELinux will block the login after reboot because the shadow file has the wrong security context.

#### 4. We need continue the booting process

We just need to run this command:

```bash
exec /sbin/init
```

It will continue the booting process. 

**Re‑enable policy**: If our environment normally disables direct root login, we can lock the account again and rely on sudo:

```bash
passwd -l root
```

## **Resetting Root Password - Method 2**

#### 1. Start VM and Stop it in GRUB menu

After opening [[01. Introduction to Boot Process in Linux|GRUB menu]], we need to press `e` from the keyboard to access configuration stuff as the first method.

#### 2. We need to add part to the config file

On the line beginning with `linux` (or `linuxefi`), we need to append a space and this:

```
rd.break
```

After the modification, we need to press `Ctrl+X` to boot. After some time, the [[05. Shell and its varieties|shell]] will open.

#### 3. We need to run the command to give read-write permission

It  will land at a dracut shell with the root filesystem mounted under **`/sysroot`** read‑only. We remount it read‑write:

```bash
mount -o remount rw /sysroot
```

**Chroot** into the installed system:

```bash
chroot /sysroot
```

#### 4. Then we can change the root password

We run:

```bash
passwd root
```

Then we need to create a specific file named `/.autorelabel` file in `/`:

```bash
touch /.autorelabel
```

That is about SELinux. Without this, SELinux will block the login after reboot because the shadow file has the wrong security context.

#### 5. At last we need to exit from the shell and let the boot continue.

We just need to run `exit` command twice:

```bash
exit
exit
```

The system will continue boot or reboot. The relabel may take several minutes.

**Re‑enable policy**: If our environment normally disables direct root login, we can lock the account again and rely on sudo:

```bash
passwd -l root
```
