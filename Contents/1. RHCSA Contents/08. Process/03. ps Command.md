#process #commands 
## **What is `ps` Command?**

There is `ps` ([[01. Introduction to Process|process]] status) command which shows running processes. These are three common variants: `ps aux`, `ps fax`, `ps ef`.

---

## **`ps aux`**

Shows all processes from all users in a "BSD-style" format. The key fields are: 
- `USER` (who owns it)
- `PID` (process ID)
- `%CPU`, `%MEM` 
- `STAT` (process state like S=sleeping, R=running)
- `RSS` is the memory the process is _actually using right now_ in RAM. This is the real number you care about.
- `COMMAND` (what's running).
- `TTY` — which terminal it's attached to (`?` means no terminal, background service). `pts` or `tty`.

---
## **`ps fax`**

**`ps fax`** Same as `aux` essentially, but adds a **tree/hierarchy view** — you can visually see which process launched which (parent → child). The `f` stands for "forest". Very useful for seeing relationships.

---
## `ps -ef`

**`ps ef`** (you wrote `-ef`) This is "System V style" (different Unix tradition). Shows similar info but replaces `STAT` with `STIME` (start time) and adds `PPID` ([[02. Parent, Child and Orphan Processes|parent]] process ID) — which tells you the PID of whoever launched that process.
