## **Introduction to Permissions**

In Linux, permissions are the rules that control who can read, write, or execute a file or directory. 

Every file and directory has three types of users attached to it: the owner (the user who created it), the group (a set of users), and others (everyone else). 

For each of these three categories, there are three basic permissions: read (r), write (w), and execute (x). 

---

## **File Permissions**

> [!Info] File Permissions
> **Read** Permission in a regular file allows us:
> - Open a file with file editors like `vi`, `nano` and read (only) its contents.
> - Reading its contents using `cat`, `more`, `less`, `head`, `tail` and more commands.
> 
> **Write** Permission in a regular file allows us:
> - Open a file with file editors like `vi`, `nano` and modify (add, update, delete) its contents.
> - Redirection to a file with `>`, `>>`, `2>`, `2>>`, `&>`.
> 
> **Execute** Permission in a regular file allows us:
> - Make a file executable (script) and allow user(s) to run it.

----

## **Directory Permissions**

> [!INFO] Directory Permissions
>**Read** Permission in a directory allows us:
> - It allows to use `ls` or `ll` command inside it.  We cannot see anything inside the directory without this permission. 
> 
> **Write** Permission in a directory allows us:
> - It allows to create or delete files/directories inside. 
> 
> **Execute** Permission in a directory allows us:
> - It means you can enter the directory with `cd` and access files inside it. 
> - Allows to modify permissions of files inside.
> - If a directory has not **execute** permission, we cannot read or create anything inside it even it has **write** permission.
> - If we run `ll` from other path for the directory who has not **execute** permission for us, it will show both an error and very limited info about the inside the directory. (only file names).
> - If a directory has **execute** permission, but not **read** permission, we can enter the directory with `cd`, cannot run `ls` or `ll` to see all files inside. However if we already know a name of any file, we can run `ll that_file.txt` and see detailed info about the file.

These permissions are shown with commands like `ls -l`, and they are changed using `chmod`, `chown`, and `chgrp`. Understanding permissions is critical in Linux because security and system stability depend heavily on who is allowed to do what.
