## **Relatonship Between Processes**

In [[01. Introduction to Linux|Linux (and Unix-like systems)]], [[01. Introduction to Process|processes]] have a **family relationship**. There are three relationship types: **parent**, **child**, **orphan**.

---
## **Parent Processes**

A parent is any process that creates another process.  It uses something like `fork()` to do this.  
*Example: when we run a command in the shell, the shell becomes the parent.*

---
## **Child Processes**

A child process is a process created by other (parent) process. It gets its own PID but also keeps the parents ID (PPID).
*Example: we type `ls` → the shell creates a child process to run it.*

----
## **Orphan Process**

An orphan is a child whose parent **dies before it finishes**. In previous versions of [[02. Introduction to Red Hat|RHEL]] if the parent of the child process dies, the child process also dies. However, in newer versions, that kind of child processes does not die. 
Instead of being “lost”, Linux gives it to a special process: **systemd (PID 1)**
This process adopts and manages all orphans. It is a new parent for them.
