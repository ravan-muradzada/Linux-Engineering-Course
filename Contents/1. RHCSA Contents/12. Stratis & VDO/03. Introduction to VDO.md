## **What is VDO?**

Virtual Data Optimizer (VDO) provides inline data reduction for Linux in the form of deduplication, compression, and thin provisioning. When you set up a VDO volume, you specify a block device on which to construct your VDO volume and the amount of logical storage you plan to present.


> [!Warning] Warning
> We need to use a disk with at least 5 GB for VDO. 

---

## **Physical Size and Virtual Size**

When we create a VDO (Virtual Data Optimizer) volume, you set two sizes:

**Physical size** — the actual disk space VDO sits on.
**Virtual size** — the size that the system _thinks_ it has available.

We set virtual size larger than physical because VDO uses **deduplication and compression** to store data more efficiently. If VDO can deduplicate 4:1, then 100GB of physical space can realistically hold ~400GB of unique-looking data. The virtual size reflects that expected capacity.

Think of it like this: a zip file holding 1GB of files might only be 200MB on disk. VDO does something similar — automatically and transparently.

**The honest warning:** this is a _promise_ based on how compressible your data actually is. If you store random or already-compressed data (like videos, encrypted files), deduplication won't help much, and you can run out of physical space while the virtual size says you're fine. That's called **over-provisioning risk**.
