## **What is DNS?**

DNS (Domain Name System) is essentially the internet's phonebook. Humans remember names like `google.com`, but computers communicate using IP addresses like `142.250.185.46`. DNS is the system ([[04. Protocols and Ports|protocol]]) that translates one to the other. There are 13 main zones and more subzones of DNS. Default port for DNS is `53`.

---

## **How it works?**

When you type `google.com` in your browser, your machine first checks its own local DNS cache — if it visited this address recently, the answer is already saved locally and the process stops there, no external communication needed.

If not found locally, the request goes to your **DNS resolver**, which is usually your ISP's DNS server or a public one like Google's `8.8.8.8` or Cloudflare's `1.1.1.1`. This resolver does the heavy lifting on your behalf.

The resolver first checks its own cache. If it doesn't have the answer either, it goes to a **Root DNS server**. Root servers don't know the IP of `google.com` directly, but they know who is responsible for `.com` domains and point the resolver there.

The resolver then asks the **TLD (Top Level Domain) server** responsible for `.com`. That server doesn't know the exact IP either, but it knows which **Authoritative DNS server** is responsible for `google.com` specifically and points there.

Finally the resolver reaches the **Authoritative DNS server** for `google.com` — this one actually has the answer and returns the IP address back to the resolver, which passes it to your machine.

Your machine now has the IP, connects to it, and your browser loads the page. The whole process typically happens in milliseconds.

----

## **Summary**

`Your machine (local cache) → DNS Resolver (ISP/8.8.8.8) → Root DNS server → TLD server (.com) → Authoritative DNS server (google.com) → IP returned back to your machine`
