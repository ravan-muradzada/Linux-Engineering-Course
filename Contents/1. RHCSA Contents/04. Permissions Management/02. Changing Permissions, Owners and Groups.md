#users #commands #files #permissions 

In Linux, `chmod`, `chown`, and `chgrp` are the main commands used to control file ownership and [[01. Introduction to Permissions|permissions]].

---
## **`chmod` Command**

`chmod` means “change mode.” It is used to change the permissions of a file or directory. You can give or remove read (r), write (w), and execute (x) permissions for the owner(u), group(g), or others(o). For example, `chmod u+x file` adds execute permission to the owner, and `chmod 755 file` sets numeric permissions (owner full access, others read and execute). This command controls _what actions_ users can perform on a file.

---

## **`chown` Command**

`chown` means “change owner.” It changes the owner of a file or directory. For example, `chown user1 file` makes `user1` the new owner. You can also change both owner and group at the same time using `chown user1:group1 file`. This command controls _who owns_ the file.

---

## **`chgrp` Command**

`chgrp` means “change group.” It changes only the group ownership of a file. For example, `chgrp group1 file` sets the group to `group1`. It is similar to `chown`, but only for the group part.
