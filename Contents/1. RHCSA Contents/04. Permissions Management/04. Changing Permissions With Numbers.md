#permissions #commands #files 

`chmod` with numbers uses **octal (base-8) values** to set [[01. Introduction to Permissions|permissions]] in a compact way. Instead of writing `u+rwx,g+rx,o+rx`, you write a three-digit number like `755`.

Each permission has a numeric value:

- read (r) = 4
- write (w) = 2
- execute (x) = 1

You add them together for each user category.

There are three positions in a number like `754`:

- First digit → owner
- Second digit → group
- Third digit → others

Example:

`chmod 755 file`

Break it down:

- 7 (4+2+1) → owner has rwx
- 5 (4+1) → group has r-x
- 5 (4+1) → others have r-x

So `755` means: full access for owner, read and execute for everyone else.

Another example:

`chmod 644 file`

- 6 (4+2) → owner has rw-
- 4 → group has r--
- 4 → others have r--

Very common for normal files.

Important mindset: numbers **replace** all permissions at once. Unlike symbolic mode (`u+x`), numeric mode does not “add” or “remove” — it sets exactly what you specify. If you forget a permission, it will be removed.
