## **What is Protocol?**

A protocol is a set of rules defining how data is communicated between two sides. It is purely theoretical — just an agreed language. HTTP defines how web data is structured, SSH defines how secure remote connections work, SMTP defines how emails are sent. The protocol itself has no physical existence.

---

## **What is Port?**

A port is a numerical door on your machine, ranging from 0 to 65535. Your machine runs many services simultaneously, and when data arrives, the operating system uses the port number to decide which service gets that data. Port is a real, practical mechanism on your machine.

----

## **Communication Between Them**

Now the connection between them — a protocol alone doesn't tell your OS where to deliver the data, and a port alone doesn't tell your OS how to handle the data. They need each other. IANA (Internet Assigned Numbers Authority) created a standardized agreement mapping well-known protocols to specific port numbers so that everyone on the internet agrees without extra configuration. HTTP goes to port 80, HTTPS to 443, SSH to 22, DNS to 53, FTP to 21.

But this mapping is purely convention, not a technical rule. You can run an HTTP server on port 9090 and it will work perfectly fine — the protocol rules remain identical. The only consequence is that clients won't find it automatically since they expect port 80, so you'd have to tell them explicitly, like `192.168.1.10:9090`.

So to summarize the relationship cleanly — protocol is the language of communication, port is the door on your machine, and the number assigned to each protocol is simply a globally agreed convention so clients always know which door to knock on without being told.

## **Commands For Ports**

To see **which ports are listening on our server**, the most common command is:
```bash
ss -tulpn
```

Meaning:

- `-t` → TCP ports
- `-u` → UDP ports
- `-l` → listening sockets
- `-p` → show which process opened the port
- `-n` → show numeric ports (no DNS/service name)

We can also use this command:
```bash
netstat -tulpn
```
