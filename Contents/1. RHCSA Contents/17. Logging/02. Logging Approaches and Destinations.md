#logging 

## **Approaches**

There are three main [[01. Introduction to Logging|logging]] approaches in [[01. Introduction to Linux|Linux]] system:

1. `systemd-journald`: With the introduction of [[01. Introduction to Daemons and Systemd|systemd]], the journald log service `systemd-journald` has been introduced also. This service is tightly integrated with systemd, which allows administrators to read detailed information from the journal while monitoring service status using the systemctl status command or the journalctl command. `systemd-journald` is the default solution for logging in [[02. Introduction to Red Hat|RHEL 9]].
2. Direct write: Some services write logging information directly to the log files—even some important services such as the Apache web server and the Samba file server. This approach to logging is not recommended.
3. `rsyslogd`: It is traditional logging system. Even if `systemd-journald` is now the default for logging, `rsyslogd` provides features not offered by `systemd-journald`, and for that reason it is still offered on RHEL 9. It is highly configurable (filter, route, forward logs) and often used for **long-term storage or sending logs to other servers**.

---
## **Log Writing Style**

There is a style of logging which is used by most loggers. Example:

```
Jan 26 09:45:15 localhost systemd[1]: fprintd.service
successfully..
```

- **Date and time**: Every log message starts with a timestamp. For filtering purposes, the timestamp is written as military time.
- **Host**: The host the message originated from. This is relevant because rsyslogd can be configured to handle remote logging as well.
- **Service or process name and PID**: The name of the service or process that generated the message.
- **Message content**: The content of the message, which contains the exact message that has been logged.

---
## **Destinations**

The main logging destination in Linux system is under `/var/log/`.  Direct writer apps and `rsyslogd` mostly writes their logs here. However, `systemd-journald` does not write their logs here, it writes them in `/run/log/journal/` which is not permanent (saved in RAM). So, when we need to make them permanent it is preferrable to put them under `/var/log`, too.


| File Destination                                                        | Contents                                                                                                                                                                    |
| :---------------------------------------------------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `/var/log/messages`                                                     | General system messages: service starts/stops, general warnings/errors, kernel-related messages (some of them), hardware events (sometimes).                                |
| `/var/log/dmesg` (viewed via `dmesg` command, not a file in most cases) | [[01. Kernel Basics\|Kernel]] ring buffer logs. Shows hardware, driver, and boot-time kernel messages like disk detection or USB events.                                    |
| `/var/log/secure`                                                       | Security/authentication logs. Very important file: [[02. Configuring SSH\|SSH]] logins, [[01. User Management\|sudo usage]], login failures, and PAM authentication events. |
| `/var/log/boot.log`                                                     | Logs from system [[01. Introduction to Boot Process in Linux\|boot process]] and startup services. Useful for checking what happened during startup.                        |
| `/var/log/audit/audit.log`                                              | Security auditing logs from **auditd**. Tracks SELinux events, file access rules, and security policy violations.                                                           |
| `/var/log/maillog`                                                      | Mail system logs. Used by mail services like Postfix or Sendmail to track email delivery, errors, and queue activity.                                                       |
| `/var/log/httpd`                                                        | Apache web server logs directory.                                                                                                                                           |
