#scripting 

## **String tests:**

```
-z "$var"    → true if string is EMPTY
-n "$var"    → true if string is NOT empty
```

---
## **File tests:**

```
-f "$path"   → true if it's a regular FILE (exists)
-d "$path"   → true if it's a DIRECTORY
-e "$path"   → true if it EXISTS (file or directory, doesn't matter)
-x "$path"   → true if it's EXECUTABLE
-r "$path"   → true if it's READABLE
-w "$path"   → true if it's WRITABLE
-s "$path"   → true if file exists and is NOT empty (has size)
```

---
## **Quick examples:**

```bash
if [[ -z "$username" ]]; then
    echo "No username provided"
fi

if [[ -f /etc/passwd ]]; then
    echo "File exists"
fi

if [[ -d /home/ravan ]]; then
    echo "Directory exists"
fi
```

---
## **Conclusion**

**Most used in real scripts:** `-z` to validate input, `-f` to check before reading a file, `-d` to check before creating a directory.

For RHCSA, `-f`, `-d`, `-e`, and `-z` are the ones that come up most. Know those well.
