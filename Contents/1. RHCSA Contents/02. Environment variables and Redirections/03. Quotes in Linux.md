#commands 

In `bash` (used in systems like Red Hat), single quotes `' '`, double quotes `" "`, and backticks 
`` ` ` `` all behave differently.

---

## **`' '` - Single Quotes**

Everything inside them is taken literally. No variable expansion, no command substitution, no special character meaning. 

Example:
```bash
echo `$variable_1`
```
The result will be just `$variable_1`, not the variable expansion.

---
## **`" "` - Double Quotes**

They allow variable expansion and command substitution, but prevent word splitting and wildcard expansion. Example:
```bash
echo "$HOME"
```
This prints your actual home directory, because `$HOME` is expanded.

---

## **`` ` ` `` - Backticks**

Now backticks `` ` ` `` are for command substitution. They execute a command and replace it with its output. We also use it in files where the commands need to be run.

Example:
```bash
echo `date`
```
This runs `date` and prints its result.
