## **What is Bash Scripting?**

Bash scripting is just writing a series of [[06. Basic Commands in Linux Terminal|Linux commands]] into a file so they run automatically, one after another, instead of you typing them manually each time.

**A simple example:** instead of running 3 commands every morning to check disk, memory, and services — you put them in one file and run that file once.

---
## **Creating and running a script:**

First, create the file:

```bash
vim myscript.sh
```

Inside the file, the first line must always be:

```bash
#!/bin/bash
```

This is called the **shebang** — it tells the system "use bash to run this file." Without it, the system doesn't know what interpreter to use.

Then write your commands below it:

```bash
#!/bin/bash
echo "Hello, Ravan"
whoami
date
```

Save and exit. Now make it executable:

```bash
chmod +x myscript.sh
```

Then run it:

```bash
./myscript.sh
```

---

## **How to Run Script File?**

There are three main methods to run the script file.

#### `./script`

We basically write the `./` and script file name, it will run. Beforehand, we need to be sure it has [[01. Introduction to Permissions|execute permission]]. 

#### `bash <FILE_NAME>`

We can use `bash` command and following the file name. In that case, we do not need to give execture permisison to the file.

#### `PATH` Variable

We need to add this script to PATH variable.
