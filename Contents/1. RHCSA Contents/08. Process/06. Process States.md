#process #theory 

A process in Linux can be in one of these states:

**R — Running** (or runnable): actively using CPU or waiting in the run queue ready to go.

**S — Sleeping** (interruptible): waiting for something (like user input or a file), but can be woken up by a signal. This is the most common state you'll see. *Example: If we run `cat` command without any argument, it just waits for it.*

**D — Sleeping** (uninterruptible): waiting on hardware I/O (usually disk). Cannot be killed or interrupted — even by `kill -9`. A process stuck in D state is a red flag.

**Z — Zombie**: the process finished, but its parent hasn't "collected" it yet. It holds no resources, just occupies a PID slot. Usually harmless unless there are many of them.

**T — Stopped**: paused, usually by Ctrl+Z or a debugger signal.

**I — Idle**: kernel threads that are sleeping but not counted toward load average (seen in newer kernels).
