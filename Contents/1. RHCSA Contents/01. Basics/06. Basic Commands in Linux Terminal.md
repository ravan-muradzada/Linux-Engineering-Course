#commands

## **`ls`**

The `ls` command lists files and directories within the filesystem, displaying detailed information about them. It is a part of the GNU core utilities package, which is pre-installed on every Linux distribution.

To get detailed info, we need to use either `ls -l`, or `ll`.

---

## **`cd`**

It changes directory to specified directory. `cd .` stays in the same place when `cd ..` sends us to parent directory. `cd -` sends us to previous directory.

---

## **`rm`**

It helps us to remove files. If we use it like `rm -r`, it can remove directories, too. Adding `-f` means force, so without any questions file(s) will be deleted.

`rm -rf /` is **extremely dangerous** in Linux. It **tries to delete every single file on your system**, including system files, programs, configurations, and user data. Running it as root will basically **wipe your Linux installation completely**, making it unusable.

Modern Linux distributions have some protections, e.g., `rm` may refuse to run with `-rf /` unless you override protections (`--no-preserve-root`), but it’s still catastrophic if forced.

💡 **Rule:** Never type `rm -rf /` unless you are 100% sure you want to destroy the system. For safe testing, always try on a disposable VM or folder.

---

## **`cp`**

This commands copies one file/folder to another directory (without removing original one). 
If we want to stay file's **mode, ownership, and timestamps**, we need to use `-p` flag which means `preserve`. Example: `cp -p source_file destination`.

---

## **`exit`, `logout`, Ctrl+D**

- We can use `exit` command to terminate current session. It works in terminal, scripts and SSH sessions.
- Specifically used to **log out of a login shell**. Example: logging out from a text-based SSH session or a console login. If you type `logout` in a **non-login shell**, it may do nothing.
- In a shell, it **signals the shell to exit** if you are at the command prompt.

## **`cat`, `more`, `less`, `head`, `tail`**

##### `cat`
Stands for **concatenate**, shows the **entire content** of a file at once.

##### `more`
Shows **one screen of text at a time**. It is useful for large files. Navigation:
- Space → next page
- Enter → next line    
- `q` → quit

##### `less`
It is similar to `more`, but more powerful, so it can scroll forward and backward. Preferred more in modern Linux. Navigation:
- Up/Down arrows → move line by line    
- Space/PageDown → next page    
- `b` → previous page    
- `/search` → search text
- `q` → quit

##### `head`
Shows the first lines of the file (default 10 lines). To show x lines from head: `head -n x file.txt`.

##### `tail`
Shows the last lines of the file (default 10 lines). To show x lines from tail: `tail -n x file.txt`.

## **`who am i`, `whoami`, `w`**

##### `who am i`
It shows **your current login session info** like 
`root     pts/0        2026-02-23 00:37 (192.168.88.1)`.

##### `whoami`
It only shows your username. Nothing more. Like `mr-ravan`.

##### `w`
Shows all logged-in users with their terminals, login time, idle time.
When we run `w` command in `TTY` command, there are might be two types of results like `tty1` and `pst/0`. Here `tty` means local terminals, but `pst` is SSH sessions or remote logins.

----

## **`uname`**

It helps us to see the kernel release version only, when we type `uname -r`.
To see detailed info, use `uname -a`.

---

## **`hostnamectl set-hostname <name>`**

It helps us to change the system hostname of our Linux machine. Basically it renames the computer. The change is permanent. To be able see the change we need to re-login or open a new bash shell. (For new bash shell run `bash`).

---
## **`df -h`, `du -sh`, `du -sh *`**

##### `df -h`
This is a Linux command that shows how much disk space is used and free on your system.

- `df` → disk filesystem info
- `-h` → human-readable (shows sizes in MB, GB instead of just bytes)

So when you run it, you see something like:

```
Filesystem      Size  Used Avail Use% Mounted on
/dev/sda1       50G   20G   28G  42% /
```

This tells you total space, used space, free space, and where each disk is mounted.

##### `du -sh`
It’s for **checking the size of a directory or file**, not the whole disk. Without any argument shows the size of current directory. 

##### `du -sh *`
It shows the size of each file in a current directory.

---
## **`free -h`**
It shows **how much RAM (memory) your system is using and free**.  Here `Mem` is RAM, `Swap` is extra memory from the disk.

---
## **`lsblk`**

It shows all storage devices attached to your system in a tree format:
**Disks** (`TYPE = disk`), **Partitions** (`TYPE = part`), their sizes, where they are mounted.
