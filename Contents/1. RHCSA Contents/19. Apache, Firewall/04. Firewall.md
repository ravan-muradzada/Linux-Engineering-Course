#firewall

## **What is Firewall?**

A firewall controls **which network traffic is allowed in or out** of your system. It sits at the network level and filters packets based on rules you define — by port, protocol, source IP, etc.

On RHEL, the default firewall tool is **firewalld**, which is a frontend that manages **netefilter** rules in the kernel. The actual filtering happens in the kernel, firewalld just makes it easier to configure.

The command that helps us during working firewall is `firewall-cmd`.
Firewall service is `firewalld.service`.

---
## **Zones**

**The core concept is zones.** Each network interface is assigned to a zone, and each zone has its own rules. For example:

- `public` zone — strict, for untrusted networks
- `trusted` zone — everything allowed
- `internal` zone — for internal network interfaces

To see all zones:

```bash
firewall-cmd --list-all-zones
```

To get the default (current) zone:

```bash
firewall-cmd --get-default
#default one is public 
```

To switch another zone:

```bash
firewall-cmd --set-default-zone=block
systemctl restart firewalld.service
#Switches to block zone
```


> [!Info] Reloading Firewall
> After any change in firewall, we need to reload it. We have two methods to reload it:
> ```bash
> systemctl reload firewalld.service
> ```
> The second method:
> ```bash
> firewall-cmd --reload
> ```
> Moreover, we can also restart `firewalld.service` with [[07. Systemctl Command|systemctl command]].
