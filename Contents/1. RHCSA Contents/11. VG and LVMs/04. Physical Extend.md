## **What is Physical Extend?**

**Physical Extent(PE)** ŌĆö it's the smallest unit of storage that LVM works with inside a [[03. Volume Group|Volume Group]].

Think of it like LVM breaking the entire VG into equal-sized chunks. By default each PE is **4MB**. When you create an LV and say "give me 1GB," LVM is actually allocating 256 PEs (256 ├Ś 4MB = 1GB) behind the scenes.

PE size matters when you need very large LVs ŌĆö if PE size is too small, LVM hits limits. You can set a custom PE size when creating the VG with `vgcreate -s`.

We can see PE size using `vgdisplay` command.

---
## **LV Creating and Extending With PE Size**

We can define the [[05. Logical Volume Management|LV]] size with PE size while creating or extending:

```bash
lvcreate -n <LV_NAME> -l 100 <VG_PATH> #Create LV with 50 LV size
lvextend -l 50 <LV_PATH> #Extend the LV 50 LV size 
```

---
## **Changing PE Size**

By default PE size is 4 MB. However, we can change it while creating VG. Our command is:

```bash
vgcreate <VG_NAME> -s <PE_SIZE> <PARTITION_NAME>
```
