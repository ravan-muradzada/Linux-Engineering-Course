## **How To Extend VG?**

It is another superior feature of [[03. Volume Group|VG]] that their size can be increased. To be able to reach that we need to have an extra [[01. Introduction to Disk|disk]] or [[02. Partitioning and Its Types|partition]]. If we have a disk, we need to partition(s) and take one.

We need to use this command:

```bash
vgextend <VG_NAME> <PARTITION_PATH>
```

When we use `vgs` command, we can see the size of VG has increased.

## **How To Shrink VG?**

You don't shrink a VG directly ÔÇö instead you **remove a PV from it** using `vgreduce`:

```bash
vgreduce vg0 /dev/sdb
```

But before removing a PV, you must move all its data to other PVs in the group:

```bash
pvmove /dev/sdb
```

Then remove it:

```bash
vgreduce vg0 /dev/sdb
```

If you also want to remove the PV label entirely from the disk:

```bash
pvremove /dev/sdb
```

**The key rule:** if `pvmove` fails because there isn't enough free space on the remaining PVs to absorb the data, you can't remove that PV. Free up space first.
