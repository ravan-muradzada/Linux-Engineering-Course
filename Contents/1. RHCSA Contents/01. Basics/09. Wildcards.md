#wildcards #commands 

## **What is Wildcards?**

Wildcards are special characters the shell uses to match filenames or paths. They are not [[08. Regular Expressions (Regex) and grep Command|regex]], but simpler patterns used by commands like `rm`, `ls`, `cp`.

| Wildcard | Meaning                                              | Example                                           |
| -------- | ---------------------------------------------------- | ------------------------------------------------- |
| `*`      | Matches **any number of characters**, including zero | `*.txt` → all `.txt` files                        |
| `?`      | Matches **exactly one character**                    | `file?.txt` → `file1.txt`, `fileA.txt`            |
| `[abc]`  | Matches **any one character** in the set             | `file[12].txt` → `file1.txt`, `file2.txt`         |
| `[a-z]`  | Matches **any character in the range**               | `[a-c]*` → `a.txt`, `b.txt`, `c.txt`              |
| `[^abc]` | Matches **any character NOT in the set**             | `file[^0-9].txt` → skips `file1.txt`, `file2.txt` |

---

## **Usage of Wildcards**

```bash
ls *.sh #Lists all files ending with ".sh"
rm file?.txt #Removes files like file1.txt, file2.txt and etc
cp data[0-9]* /backup/ #Copies the files like data0, data1 and etc to backup dir.
rm F_[abc].txt #Removes only F_a.txt, F_b.txt, F_c.txt.
```
