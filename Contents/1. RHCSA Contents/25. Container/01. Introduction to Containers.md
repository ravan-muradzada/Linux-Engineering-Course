#containers

A container is a **lightweight, isolated package** that runs an application along with everything it needs — its code, libraries, config files, dependencies — all bundled together.

Think of it this way: imagine you write a Python app on your laptop. It works perfectly. But when you move it to a server, it breaks because the server has a different Python version or missing libraries. A container solves this by carrying its own environment with it — it runs the same way everywhere.

**How is it different from a VM?**

A VM virtualizes the entire hardware and runs a full OS on top. A container shares the host OS kernel but isolates the application's environment. This makes containers much faster to start and far lighter in resource usage.

**Docker vs Podman** — both run containers, but Podman is daemonless (no background root service running) and is the default tool in RHEL.
