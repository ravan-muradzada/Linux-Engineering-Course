## **What is Package Manager?**

A **package manager** is a tool that installs, removes, updates, and manages software on a Linux system. In Linux, software is usually distributed as **packages**. A package contains the program itself plus extra information such as version, dependencies, configuration files, and scripts needed during installation. Instead of manually copying files to the system, the package manager handles everything: it places files in the correct locations, checks dependencies, and keeps a database of installed software.

---

## **RPM**

In **Red Hat–based systems** (like RHEL, Rocky, Alma, Fedora), the basic package format is **RPM**.

**RPM (RPM Package Manager)** is the low-level tool used to install and manage `.rpm` packages directly. It can install, remove, query, and verify packages. For example, administrators can check if a package is installed, view its files, or install a package file manually. The problem is that RPM **does not resolve dependencies automatically**. If a program needs other libraries or packages, RPM will fail and ask you to install those dependencies yourself. This becomes difficult when many dependencies are involved. RPM also **does not automatically download packages from repositories**, so you usually need the `.rpm` file already on your system.

Because of these limitations, higher-level tools were created.

---

## **YUM**

**YUM (Yellowdog Updater Modified)** was designed to make package management easier. It works on top of RPM but adds important features. YUM can automatically **resolve and install dependencies**, **download packages from repositories**, **update the system**, and **manage groups of packages**. Instead of manually finding required libraries, YUM downloads everything needed.

---

## **DNF**

Later, **DNF (Dandified YUM)** replaced YUM in modern Red Hat systems (RHEL 8 and later). DNF performs the same role but is **faster, more efficient, and has better dependency resolution**. It also has a cleaner codebase and improved performance when working with large repositories. 

> [!Info] DNF and YUM
> In modern systems **DNF** is main package manager to use.  Actually, `yum` command is actually just a compatibility link to `dnf`. So when we run `yum` command, it is not real `yum`, it is just `dnf`.
