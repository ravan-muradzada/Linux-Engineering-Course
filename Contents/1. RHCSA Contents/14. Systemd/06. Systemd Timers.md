#systemd #timers

## **What is Timer?**

A systemd timer is basically **[[01. Introduction to Crontab|cron]] but built into [[01. Introduction to Daemons and Systemd|systemd]]**. It schedules a service to run at specific times or intervals.

Every timer has two files — a `.timer` [[02. Units|unit]] and a `.service` unit with the same name. The timer is the schedule, the service is the actual job that runs.

---

## **Main Lines**

**`[Unit]`** section is same as always, just a description.

**`[Timer]`** is the important section:

- `OnCalendar=` — run at a specific time, like cron. For example `OnCalendar=*-*-* 02:00:00` means every day at 2am. This is the most common one. Syntax: `DayOfWeek Year-Month-Day Hour:Minute:Second`

- `OnBootSec=` — run once after boot. `OnBootSec=10min` means 10 minutes after system starts.

- `OnUnitActiveSec=` — run repeatedly after the last time it ran. `OnUnitActiveSec=1h` means every hour after the last run.

- `Persistent=true` — if the system was off when the timer was supposed to fire, run it immediately on next boot. Very useful, this is what anacron used to do.

**`[Install]`**

- `WantedBy=timers.target` — same idea as `sockets.target`, just groups all timers together.

---

You manage timers just like services:

```bash
systemctl enable --now myjob.timer
systemctl list-timers          # see all timers and when they last/next ran
```

The key thing to remember — **you enable and start the `.timer` file, not the `.service` file.** The timer triggers the service automatically.
