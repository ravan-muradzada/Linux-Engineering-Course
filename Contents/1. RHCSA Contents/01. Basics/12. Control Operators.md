#operators

## **What is Control Operators?**

In the Linux command line (like in `bash`), `;`, `&&`, and `||` are called **command operators** or more specifically **control operators**. They control how multiple commands are executed in relation to each other. This is very important in real administration because you often chain commands together instead of running them one by one.

---

## **What is `;` ?**

The semicolon means to do each command no matter what happens. So even if the previous command(s) has failed/passed, it continues and tries to run the next one.

Example:
```bash
cp file.txt dest/ ; pwd ; ls
```

Here even `cp` command fails, it will continue and run the other ones.

---
## **What is `&&`?**

The double ampersand means "run the next command only if the previous ones does not fail". So it runs all commands up to the failed one.

Example:
```bash
pwd && ls && cp file.txt dest/
```

If no one fails, all will be run.

---
## **What is `||`?**

The double pipe means "run the next command only if the previous one has failed". So it tries to run the commands up to the succeeded one.

Example:
```bash
wrongCommand_1 || wrongCommand_2 || pwd || ls
```

Only `pwd` command will run, because previous ones have failed, it reaches `pwd` and stops here.
