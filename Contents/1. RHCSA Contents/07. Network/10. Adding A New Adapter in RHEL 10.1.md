#network #commands #configuration #rhel-10

We have learned how to [[09. Usage of nmtui|add a new adapter]] and change its config via `nmtui`. However, the settings have changed a little in rhel 10. So we need to reanalyze process again.

It is the same to add a new network adapter via VMware Workstation Pro. However, problem occurs when we use `ip a` to see all adapters. For our new adapter, it shows:

```bash
4: ens224: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc mq state UP group default qlen 1000
    link/ether 00:0c:29:8b:9a:9f brd ff:ff:ff:ff:ff:ff
    altname enp19s0
    altname enx000c298b9a9f
```

As you can see, it does not give it a default private ip with [[08. DHCP and DORA|DHCP]]. Moreover, if we use `nmtui` we cannot see its name in the list, so we cannot give it a private ip manually. 

---

## **Solution**

The solution is very easy:

```bash
nmcli device connect <ADAPTER_NAME>
nmcli connection modify <ADAPTER_NAME> connection.autoconnect yes
```

We can see the adapter name when we type `ip a`. After that, it will have a private via from DHCP and be visible in `nmtui`.
