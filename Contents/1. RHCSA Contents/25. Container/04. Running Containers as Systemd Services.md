#containers #systemd #services 

## **Create User**

First we need to create a new user and set a password for him:

```bash
useradd linda
passwd linda
```

---
## **Directory for Mount**

If we want to mount some directory in container, we create one in VM:

```bash
mkdir /container-dir
chown linda:linda /container-dir
chmod o+w /container-dir
```

Here we should also set its user and group owner as the created user. Moreover, it is recommended to give others [[01. Introduction to Permissions|write permission]].

---
## **Linger Feature Enable**

We run this command to enable the linger feature for user `linda`:

```bash
loginctl enable-linger linda
```

This command tells systemd to keep Linda's user session alive **even after she logs out**.


> [!Info] Importance
> It's commonly used for service accounts or users who need to run persistent user-level services without being root — for example, a developer running a user-scoped container or timer that should keep running overnight.

---
## **Switching to newly created user**

We should not switch to the newly created user using `su -` command. There is only one way to ensure everything works well. We should break the current [[02. Configuring SSH|SSH]] connection and create a new one:

```bash
ssh linda@<IP>
```

---
## **Podman Login**

If we want to get the images from specific website, we use login:

```bash
podman login
```

---
## **Pull Image and Create Container**

We should pull the image:

```bash
podman pull nginx
```

Then we create a container:

```bash
podman run -d --name my-cont -p 8080:80 nginx
```

---
## **Systemd Service Directory Creating**

We should create this directory and switch to it:
```bash
mkdir -p ~/.config/systemd/user
cd ~/.config/systemd/user
```

---
## **Creating Service File**

We use this command to create a service for our container:

```bash
podman generate systemd --name <CONTAINER_NAME> --files
```

We use these commands afterward:

```bash
systemctl --user daemon-reload #Ensure that systemd picks up the changes.
systemctl --user enable container-mynginx.service #Just enable, no start
```

At last we reboot VM.
