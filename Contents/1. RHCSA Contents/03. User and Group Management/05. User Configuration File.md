#users #configuration #files

## **`/etc/login.defs`**

`/etc/login.defs` is a **system-wide configuration file** that controls default behavior for user accounts, password policies, and UID/GID ranges. 

We can find user related settings with the command:

```bash
chage -l username
```

----
## **Default ID Range**

```bash
#
# Min/max values for automatic uid selection in useradd(8)
#
UID_MIN                  1000
UID_MAX                 60000
# System accounts
SYS_UID_MIN               201
SYS_UID_MAX               999
```

---

## **Password Aging Settings**

```bash
PASS_MAX_DAYS   99999
PASS_MIN_DAYS   0
PASS_WARN_AGE   7
```

If we did not give them manually, these are default values. We can check users' password settings from `/etc/shadow`. [[02. Users and Passwords Files|Users and Passwords]]

---

## **Other Settings**

```bash
UMASK  022
USERGROUPS_ENAB yes
CREATE_HOME     yes
```

Here if the first setting is enables (`yes`), whenever a new user is created, the system creates a primary group with his name and this user to the group. That is default feature. However, we can make it `no`, then its primary group will be `users` [[04. User Default Settings|group]] with group id `100`.
[[05. UMask Command|UMASK]] value is also here.
