#configuration #files

## **Main Config Files**

There are four main bash configuration files in Linux system:
`/etc/bashrc`, `/etc/profile`, `~/.bashrc`, `~/.bash_profile`.

The files inside `/etc` folder is for all users in the system. The others are different for each user and located in users' home directories.

---

## **Bashrc and Bash Profile Differences**

**Bash Profile** files run only when a login shell starts. If we change **bash profiles** files after login process, running `bash` command (creating a non-login shell) is not enough. They do not work with subshells. To run the files we should re-login, or just type this command: `source ~/.bash_profile` or `source /etc/profile`. 

**Bashrc** files normally run only when a nonlogin shell starts, like using `bash` command. Normally, they do not run when a login shell starts. However in **bash profile** files there are a code line where **bashrc** files are run from here. Thus even if **bashrc** files themselves do not run in login shell, from **bash profile** files they run. So **bashrc** run for both login and nonlogin shells. We also run **bashrc** files using these commands: `source ~/.bashrc` or `source /etc/bashrc`.

---

## **`source` Command**

As you can see to run the bash config files we need whether to re-login or typing `bash` command. However `source` command helps us a lot, so we do not need to re-login or type `bash`.

---

## **Usage of Files**

| File              | Scope       | When it runs      | What to put                                     |
| ----------------- | ----------- | ----------------- | ----------------------------------------------- |
| `/etc/profile`    | system-wide | login shell       | environment variables, PATH, umask, locale      |
| `/etc/bashrc`     | system-wide | interactive shell | aliases, functions, prompts                     |
| `~/.bash_profile` | per-user    | login shell       | user-specific environment                       |
| `~/.bashrc`       | per-user    | interactive shell | user-specific aliases, functions, shell options |

---

## **`export` Keyword**

`export` keyword is used in **bashrc** or **bash profile** files with variables. It means:
*“Make this variable part of the environment so child processes can see it.”*

Without it variables exist only inside the current shell. If you start another program (like `bash`, `vim`, `python`, etc.), that program will **not** see them.
