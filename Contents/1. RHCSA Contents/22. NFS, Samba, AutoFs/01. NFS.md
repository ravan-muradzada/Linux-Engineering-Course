#nfs #packages

## **What is NFS?**

NFS (Network File System) lets one [[01. Introduction to Linux|Linux]] machine **share a directory** over the [[01. IP Address and Its Variations|network]], and other machines **[[05. Mounting|mount]] it** as if it were a local folder.

**Two roles:**

- **Server** — has the actual files, _exports_ a directory
- **Client** — connects and _mounts_ that export, accesses files transparently

**How it works (simply):**

When you do `ls /../shared` on the client, NFS silently sends that request over the network to the server, gets the result, and returns it to you. You don't feel the difference.

## **Server Side**

#### Package
We need to download this [[01. Introduction to Package Management|package]]:
```bash
dnf install nfs-utils -y
systemctl enable --now nfs-server
```

#### Firewall
We need to add its [[03. Systemd Services|service]] file to [[04. Firewall|firewall]]:
```bash
firewall-cmd --add-service=nfs --permanent
firewall-cmd --reload
```

NFS doesn't work alone — it's built on top of **RPC (Remote Procedure Call)**, which is a system that lets a program on one machine call a function on another machine over the network. It uses `rpc-bind` and `mountd`, too. So, we should add these services to firewall, too:

```bash
firewall-cmd --add-service=rpc-bind --permanent
firewall-cmd --add-service=mountd --permanent
firewall-cmd --reload
```
#### Sharing Config
To make one directory shared, we need to go `/etc/exports` file and add a line there:
```conf
/nfs-shared   *(rw,no_root_squash)
```

- `/nfs-shared` - It is the directory path which we want to share.
- `*` - It is who can access it. When we write `*`, everyone in the same network can access it. This can be a single IP (`192.168.1.10`), a subnet (`192.168.1.0/24`) and also a hostname (`client1.example.com`), or a wildcard (`*.example.com`).
- `(rw,no_root_squash)` - These are options. Comma-separated, no space between client and the opening parenthesis (that space matters — a space before `(` means the options apply to _everyone_, not just that client).


> [!Info] Root Privilage of Client
>  By default, NFS has a security behavior called **root squash** — if a client connects as `root`, the server secretly treats them as `nobody` (an unprivileged user). This prevents a client's root from having root power over the server's files. 
>  **`no_root_squash` turns that protection off.** So if you're root on the client, you're also root on the server's shared files.


> [!Warning] `ro` and `rw`
> When the server sets `ro`, the client can read files but any write attempt gets blocked — the kernel returns a "Read-only file system" error.
> 
> When the server sets `rw`, the client _can_ write — but there's a second layer to think about. The server's normal Linux **[[01. Introduction to Permissions|file permissions]] still apply**. So even with `rw` in exports, if the actual file on the server is owned by root with `600`, a different user can't write to it. `rw` just _allows_ the NFS layer to pass write requests through — it doesn't override filesystem permissions.
> 
> So think of it as two gates: NFS options are the first gate, file permissions are the second. Both have to allow it for a write to succeed.

We should also determine the file permissions of the directory. The devices via network is `others` for the directory of server. So, when we want to add or remove some permissions to this directory, we will do it based on `others`.
It is recommended to add [[03. Sticky Bit|sticky bit]] permission to this directory (in server):
```bash
chmod o+t <SHARED_DIR_NAME>
```
#### Service Start (and enable)

Then we need to enabe and start the service of `nfs`:

```bash
systemctl enable --now nfs-server.service
```

## **Client Side**

#### Package
We should download the same package here, too:
```bash
dnf install nfs-utils -y
systemctl enable --now nfs-server
```

#### Show Mounts

Now we can look at the directories shared here with this command:

```bash
showmount -e <SERVER_IP>
```

#### Mounting in Client

In Client Side we should mount the shared directory to be able to use it. We can create a directory with any name and mount it.

**Temporary Mount**:
`mount <SERVER_IP>:/<SHARED_DIR_NAME>  /nfs-dir`

**Permanent Mount**:
We should add this line to [[01. Introduction to fstab file|/etc/fstab]] file. Moreover, it is good to use [[02. Syntax of fstab file|_netdev]] here.
`<SERVER_IP>:/<SHARED_DIR_NAME>   /<MOUNTPOINT>  nfs  defaults,_netdev   0 0`

Afterward we can work with this directory.
