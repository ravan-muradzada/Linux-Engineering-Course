## **What is Priority and Nice Value?**

Priority in [[01. Introduction to Process|processes]] refers to the importance level assigned to a process, determining how much CPU time it receives compared to other processes. It is between `0` and `39`.

The nice value is a specific number that indicates a process's priority in Unix-like systems, ranging from `-20` (highest priority) to `19` (lowest priority), where a higher nice value means the process is "nicer" to others by yielding CPU time more readily.

`Priority` = `Nice + 20`

*A process with `Nice = 0` and `Priority = 20` → normal priority.*

---

## **How to Set Nice Value?**

In Linux we cannot directly change priority value. Instead, we change nice value, and the system update its priority value for the process. There are two commands to change nice value:

##### `nice` Command

```bash
nice -n 10 sleep 1000 &
```

This command sets nice value when we start the process. It only works **at process creation**.

##### `renice` Command

```bash
renice -n 10 -p 1234
```

This command changes the nice value of existing process. We can increase/decrease priority while it runs. Here, `-p 1234` is PID of running process.
