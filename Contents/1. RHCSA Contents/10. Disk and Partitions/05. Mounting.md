## **What is Mounting?**

We have talked about [[02. Lsblk, Mounting and Mountpoint|mounting]] in previous notes. However, it was not deep enough, now we need to dive deeply.

**What is mounting?** After creating a [[04. Filesystem|filesystem]], [[01. Introduction to Linux|Linux]] still can't access it until you attach it to a directory. That directory is called a **mount point**. Mounting is just connecting a [[02. Partitioning and Its Types|partition]] to a directory so you can read and write to it.

---
## **Commands to Mount (Temporary)**

We can mount a partition to a directory using `mount` command:

```bash
mount /dev/nvme0n2p1 /data
```

That's it. Now everything you put in `/data` goes to `/dev/nvme0n2p1`.

**To unmount:**

```bash
umount /data
```

*Note — it's `umount`, not `unmount`. Common typo.*

----
## **Commands to Mount (Permanent)**

**The problem with temporary mounts** — they disappear after reboot. For [[01. Introduction to fstab file|permanent mounts]] we need `/etc/fstab`.

```bash
/dev/nvme0n2p1          /data                   xfs     defaults        0 0
```

- In the first section we need to write the [[01. Introduction to Disk|disk]] path or its UUID.
- In the second section we need to write the directory where we want to mount.
- In the third section we need to write the filesystem we determined for the partition.
- In the fourth section we write `defaults`.
- In the fifth section we write `0 0`.

We will talk about the last two sections in fstab lesson.


> [!Info] First Section with UUID
> In previous example I have used partition name in fstab file. However, it is better to use partition UUID instead. 
> 
> **How to get UUID of any partition?** 
> ```bash
> blkid | grep "nvme0n2p1"
> ```
> `blkid` command shows all partitions with their UUID. If we [[08. Regular Expressions (Regex) and grep Command|grep]] it, we can find UUID of our partition. Then we just need to copy it and paste to fstab like that:
> ```
> UUID=7dead2f9-a78b-42c9-93ba-798837983031  /data   xfs   defaults   0 0
> ```

We need to reboot or use these commands to apply the changes in fstab:

```bash
systemctl daemon-reload   # tell systemd fstab changed
mount -a                  # apply the changes now
```

*If we just reboot, nothing is needed to run to apply the changes*.


> [!Danger] Fstab Mistake
> If we make any mistake in fstab file in terms of syntax, it causes crash when we try to reboot our system. So thus we should be very careful while writing any data here. Moreover, we need to use `systemctl daemon-reload` and `mount -a` commands after the changes.

## **How to Look Mounts?**

There are mutliple ways to see mounts:

- `df -h` — It shows us disk usage in the system and mounts for all partitions.
- `lsblk` — It shows us disks and their partitions in the system, here we can also see the mounts.
- `mount` — If we just run this command, it shows all mounts. To search specific one, we run `mount | grep "nvme0n2p1"`.
