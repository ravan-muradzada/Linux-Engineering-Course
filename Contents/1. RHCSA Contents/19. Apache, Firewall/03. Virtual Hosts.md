## **What is Virtual Host?**

Virtual hosting lets you run **multiple websites on a single server** using one [[01. IP Address and Its Variations|IP]] address.

Without it, one server = one website. With it, you can host `siteA.com`, `siteB.com`, and `siteC.com` all on the same machine. When a request comes in, Apache reads the `ServerName` from the request header and routes it to the correct `DocumentRoot`.

We create and manage Virtual Hosts from `/etc/httpd/conf.d`. So, we create `*.conf` file here and create virtual host inside it.

***Here, the identifier of each website it its `ServerName` part.***

---
## **Syntax**


```conf
<VirtualHost *:80>
	ServerAdmin webmaster@account.example.com
    ServerName mysite.local
    DocumentRoot "/var/www/mysite/html"

    <Directory "/var/www/mysite/html">
        AllowOverride None
        Require all granted
    </Directory>
</VirtualHost>
```

- `<VirtualHost *:80>` ─ Opens a virtual host block. Listen on all IPs (`*`), port 80.
- `ServerAdmin` ─ Contact email shown in Apache error pages when something breaks. Not functional, just informational.
- `DocumentRoot` ─ Where this site's files live. Apache looks here when serving content for this domain.
- `<Directory>...</Directory>` ─ Allows the specified directory for apache to use. It is good to put there at least `index.html`.

We need to restart `httpd` service after creating or changing virtual host.
