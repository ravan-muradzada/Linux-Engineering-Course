## **Introduction**

We have learned about [[04. Filesystem|filesystem]] and [[05. Mounting|mounting]] in previous lessons. We created a filesystem on a [[02. Partitioning and Its Types|partition]] and mount it to any directory. Now we need to create filesystem on [[05. Logical Volume Management|LV]]. 

---
## **Filesystem**

We need to run this command:

```bash
mkfs.xfs /dev/vg01/lv01 #with xfs (filesystem type)
```

----
## **Mounting**

To temporary [[05. Mounting|mount]] we just run `mount /dev/vg01/lv01 /directory`. However, if we want to mount permanently, we need to add it to `fstab` file. 

We add this line to `fstab` file:

```bash
/dev/vg01/lv01    /data     xfs    defaults   0  0 
```

Then we run:

```bash
systemctl daemon-reload
mount -a
```



> [!Info] First Section with UUID
> In previous example I have used partition name in `fstab` [[01. Introduction to fstab file|file]]. However, it is better to use partition UUID instead. 
> 
> **How to get UUID of any partition?** 
> ```bash
> blkid | grep "nvme0n2p1"
> ```
> `blkid` command shows all partitions with their UUID. If we [[08. Regular Expressions (Regex) and grep Command|grep]] it, we can find UUID of our partition. Then we just need to copy it and paste to fstab like that:
> ```
> UUID=7dead2f9-a78b-42c9-93ba-798837983031  /data   xfs   defaults   0 0
> ```
