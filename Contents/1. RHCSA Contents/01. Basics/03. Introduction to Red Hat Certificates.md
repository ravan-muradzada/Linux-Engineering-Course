## **Red Hat Certificates**

Red Hat certifications are practical, performance-based exams that prove you can administer real Linux systems, not just answer theory questions. They are issued by Red Hat and focus mainly on Red Hat Enterprise Linux (RHEL). You sit in front of a live system, configure it, and if it works correctly, you pass. No multiple choice. This is why they are respected.

The main certification path is structured like this:

#### **RHCSA**

*First level: Red Hat Certified System Administrator (RHCSA).*  
This is your foundation. You must know Linux basics deeply and practically: user and group management, permissions, SELinux basics, networking configuration, package management (dnf), systemd services, storage management (LVM, partitions, mount points), file systems, boot process, firewall basics, SSH, and basic troubleshooting. You must be fast and accurate. No guessing. For RHCSA, you need strong command-line confidence. If you hesitate with basic commands, you are not ready.

#### **RHCE**

*Second level: Red Hat Certified Engineer.*  
This builds on RHCSA. Here the focus shifts heavily to automation and services. Modern RHCE is strongly based on Ansible automation. You must know how to write playbooks, manage inventories, configure services automatically, deploy web servers, manage users via automation, and troubleshoot configurations. This is more advanced and assumes you are already solid in RHCSA-level tasks.

#### **RHCA**

*Third level: Red Hat Certified Architect.*  
This is expert level. It requires multiple additional specialty exams (security, OpenShift, clustering, performance tuning, etc.). This is for senior engineers.
